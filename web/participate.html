Reporting bugs

RedNotebook uses launchpad for development and issue tracking. If you have found a bug in RedNotebook, please head on over to the issue tracker and create a new issue. If you don’t know how to use that site, a simple mail is equally fine. In both cases, please post the two files “configuration.cfg” and “rednotebook.log” from your “$HOME/.rednotebook/” directory.

  • Report a new bug

  • Launchpad translations

Hacking

Development on RedNotebook happens in the trunk bazaar branch. Check out the code by running:

bzr branch lp:rednotebook
Start RedNotebook by running: 
$ cd rednotebook
$ sudo apt-get install python-yaml python-webkit python-chardet
$ python rednotebook/journal.py
